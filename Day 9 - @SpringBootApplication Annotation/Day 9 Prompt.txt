1.@ComponentScan

What is Component Scanning in Spring Boot Application - Explain with Definition and Explanation 
then let's dive to our application setup with SpringBootApplication base where application Starts and ApplicationContext (Spring IOC Container) created actual start point of the application.

when application starts ApplicationContext starts IOC Container Initiliazed,then it will start looking for beans to initialize across the application, it looks at each and every java class in the application for the meta data or annotations to create the beans.

but where to look at the application like which package to see to create the beans is defined by component Scanning.

Use of ComponentScan Annotation:
which package to look at for the Bean creation and initialization. Where to look for the bean.

but when we look in our application we don't see any ComponentScan Annotation, ??

there comes our Main Application where we start - @SpringBootApplication Annotation

@SpringBootApplication is Combination of (Configuration + EnableAutoConfiguration + ComponentScan) Annotations.

No Explicit definition of ComponentScan Required beauty of Spring.
ComponentScan - scans all the Sub packages in the package where our main application resides.

Ex: Our Main Application Resides inside the Ecommerce Package in which Service,COntrollers etc all are available as sub packages and there is another package next to ecommerce that package also has sub packages with beans and when we start our application beans and components are scanned and initialized in ecommerce package and sub packages not the other package because our main application with @SpringBootApplication Resides in this Ecommerce Package.

Now our problem is how to Tackle thiss???

how to make spring to scan the other package aswell and initliaze the beans.
Sol: Add @ComponentScan(basePackages = {"com.project.ecom","com.project.ecomPayment"})
below the @SpringBootApplication Annotation.

and now if we run the application then both the packages are being scanned and beans are getting initialized

Another option just
Add @ComponentScan(basePackages = "com.project")
scans all the packages in the main package of project and  now if we run the application then both the packages are being scanned and beans are getting initialized.

Another option instead of mentioning in the main application - startpoint we can use Config.

let's create A Config class and Add @ComponentScan(basePackages = {"com.project.ecomPayment"})
and run the application we see ecomPayment not initialzed only ecom is initlaized that too beacse of main application is taking care of, this config is not working.

Sol: ComponentScan Annotation should be used along with the Configuration Annotation.


 Add 
@Configuration
@ComponentScan(basePackages = {"com.project.ecomPayment"})
and run the application

now, scans all the packages in the main package of project and  now if we run the application then both the packages are being scanned and beans are getting initialized.

next will see Exclude filters, add those classes which we don't want to scan and initialize with example clearly.
