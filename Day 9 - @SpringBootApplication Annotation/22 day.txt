VIEWS VS STORED PROCEDURES

VIEW :
SAVED QUERY (LIKE A VIRTUAL TABLE)
you can select from it as if it was a real table
mostly used for simplifying complex select queries
cannot contain control logics (loops,condition)
cannot accept parameters
always returns a result set
simplifying complex selectqueries


stored procedure

a saved block of sql code ( can include multiple queries)
can contain insert,update,delete,select
can include logics (If,case,loo,conditions)
can accept parameters (input/otput)
may or may not return a result set
