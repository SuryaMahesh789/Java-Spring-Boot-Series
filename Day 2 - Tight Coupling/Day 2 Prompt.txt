Tight Coupling,Loose Coupling,Dependency Injection
Helps to convert Tight coupling to Loose Coupling
In any project, we use multiple classes, and one object is used inside another class viceversa
one class uses members of another class for this we are creating objects inside another class because of this causing tight coupling, instead of creating the object of class 2 inside class 1 causes tight coupling to avoid this instead of creating objects then and there we need to push the object or create object at runtime 

Scenario based :

Let's say if we want to buy a mobile and when we head to mobile store and ask for a mobile, if the person says that this mobile allows only one telecom provider example only airtel is supported for this mobile and this actually a tight coupling because we don't have the freedom of selecting the telecom provider or sim let's say if some prices drop or hikes or that telecom provider is stops the service then that device is non useful makes no sense, so to remove this tight coupling we ask for a mobile which just has sim slot where we have the choice or freedom of putting whatever the sim we like to put and whenever we like we can insert,


this analogy holds true for this dependency injection concept this is a concept which helps to make tight to loose coupling this procedure enforces us to create or send the objects from outside Field,Constructor,Setter injection we can insert instead of creating the object with new keyword.


Code : example

Mobile is one class
Airtel Sim,Jio Sim,Idea sim is seperate classes 
inside mobile class we need to use Sim 

tight coupling is like creating Airtel sim inside the mobile class with new keyword no freedom of changing or choicing the remaijing sim classes


loose coupling is something like 

Mobile is one class
Sim is an interface and
Airtel Sim,Jio Sim,Idea sim is seperate classes impelements Sim Interface
inside mobile class we need to use Sim Class and we can select the class to create airtel,idea or jio whatever is required and our choice can be given thru setter,constructor injection new airtelsim() or new ideasim() or new JioSim() will be assigned from outside when mobile class is instantiated that time we will pass the selected sim class

this way of having or using interface helps us to add one layer of abstraction to the methods instead of using classes directly.
